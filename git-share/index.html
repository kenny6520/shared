一个小问题

add . 和 add --all 有什么不一样?



处理档案不同

  • 1.x 版本时候
    • git add --all 会处理 新增/修改/删除 的档案
    • git add . 则只会处理 新增和修改的档案...
  • 2.x版本以后
    • add . 和 add --all 就没有这个问题了, 都可以出现 新增/删除/修改的 档案了.

另外一点

  • 如果你使用 git add --all 不管在档案的哪一个 目录 都是有效的
  • 而 git add . 只处理当前目录 和 子目录档案

对分支的误会

你想象中的分支是什么样子?

有点人可能会认为分支是把档案 复制一份出来修改, 改完以后合并回去...事实上并不是如此

为什么大家都说在 git 里面开分支很便宜?


  • 其实分支只是只想某一个 commit 的一个指标
  • 类似于一个标签 或者是 一个贴纸...
  • 当你不消息删除了某个分支, 或者reset --hard去到某个地方, 其实原来的东西并没有消失, 只是用另外一种格式打包/ 隐藏起来你看不到了而已
  • 所谓的分支其实就只是一个 40个子元的档案而已...
  • 举个栗子😂
